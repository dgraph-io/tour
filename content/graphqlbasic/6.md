The logical connectives `and`, `or` and `not` combine multiple functions in a
filter.

Multiple parameters in a single filter argument combine with `and` logic by
default:

```
{
  getPerson(xid: "alice") {
    xid
    friends(filter: {
      xid: { eq: "dave" }
      age: { eq: 35 }
    }) {
      xid
      age
    }
  }
}
```

Each parameter in an argument object must be unique. Wrap arguments in logical
lists using `and` and `or`:

```
{
  getPerson(xid: "alice") {
    xid
    friends(filter: {
      and: [
        { age: { ge: 30 } },
        { age: { lt: 40 } }
      ]
    }) {
      xid
      age
    }
  }
}
```

Dgraph also generates inline `and` and `or` filter parameters. When combining
more than two filters or using multiple logical combinators, logical lists
produce more readable queries.

```
{
  getPerson(xid: "alice") {
    xid
    friends(filter: {
      age: { ge: 35 }, or: { xid: { eq: "frank" } }
    }) {
      xid
      age
    }
  }
}
```

Add inequality filters with the `not` logical combinator:

```
{
  queryPerson(filter: { not: { age: { le: 30 } } }) {
    id
    name
    ownsPets {
      id
      name
    }
  }
}
```

{{% notice "note" %}} The `not` logical wrapper cannot contain a list of
arguments `not: [{ age: ... }, ...]`. But it can contain multiple arguments and
other logical combinators, e.g.: `not: { and: [{ age: ... }, ...] }`
{{% /notice %}}
