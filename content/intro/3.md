With the schema in place, load some data.

Press the **run** button again and check that the response panel indicates the data was
loaded successfully.

Later lessons explain the syntax in detail.

{{% notice "note" %}}
**Only run this mutation once.** Blank nodes like `_:michael` are temporary
labels — Dgraph assigns a new UID for each one every time you run the
mutation. Running it again will create a second set of nodes (two Michaels,
two Amits, etc.) rather than updating the existing data. If that happens, run
`make restart` to reset the database and restart the tour, then
[click here](/intro/1) to begin this section again.
{{% /notice %}}

The [DQL](https://docs.dgraph.io/dql/) [RDF data format](https://docs.dgraph.io/dql/dql-rdf)
used for the [mutation](https://docs.dgraph.io/dql/dql-mutation) on the right is just one 
of the data formats our [clients](https://docs.dgraph.io/clients/), the [Ratel UI](https://docs.dgraph.io/ratel/),
and API endpoints support when loading and modifying data. [JSON](https://docs.dgraph.io/dql/json-mutation-format) and [GraphQL mutations](https://docs.dgraph.io/graphql/mutations/mutations-overview)
can be used as well.
