Attach a language tag to a string value in the mutation:

```
_:myID <an_edge> "something"@en .
_:myID <an_edge> "某物"@zh-Hans .
```

Then request a specific language on the [edge in a query]({{< relref "basic/4" >}}).

You can express the same mutation in JSON via the clients, cURL, or Ratel UI.

See the JSON:

```
 {
        "set": [
            {
                "uid": "_:myID",
                "an_edge@en": "something",
                "an_edge@zh-Hans": "某物"
            }
        ]
    }
```

{{% notice tip %}} Comparing the JSON to the RDF above clarifies both
formats. {{% /notice %}}
